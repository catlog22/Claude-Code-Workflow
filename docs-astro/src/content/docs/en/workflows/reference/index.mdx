# Workflow Reference

Detailed reference for each CCW workflow. For an overview and selection guide, see the [Workflow Introduction](/en/docs/workflows/introduction).

## Workflows by Type

### Execution

| Workflow | Complexity | Description |
|----------|-----------|-------------|
| [Lite-Lite-Lite](/en/docs/workflows/reference/lite-lite-lite) | Low | Instant execution for tiny changes (no artifacts). |

### Planning

| Workflow | Complexity | Description |
|----------|-----------|-------------|
| [Lite-Plan](/en/docs/workflows/reference/lite-plan) | Low | Lightweight planning for clear, single-module work. |
| [Plan](/en/docs/workflows/reference/plan) | Medium | Standard session planning for multi-module development. |

### Debugging

| Workflow | Complexity | Description |
|----------|-----------|-------------|
| [Lite-Fix](/en/docs/workflows/reference/lite-fix) | Low | Bug diagnosis and fix with lightweight artifacts. |
| [Debug-with-File](/en/docs/workflows/reference/debug-with-file) | Medium | Hypothesis-driven debugging with documented understanding. |

### Analysis

| Workflow | Complexity | Description |
|----------|-----------|-------------|
| [Multi-CLI-Plan](/en/docs/workflows/reference/multi-cli-plan) | Medium | Collaborative planning with multiple CLIs/perspectives. |
| [Analyze-with-File](/en/docs/workflows/reference/analyze-with-file) | Medium | Collaborative analysis with exploration + recorded conclusions. |

### Testing

| Workflow | Complexity | Description |
|----------|-----------|-------------|
| [TDD-Plan](/en/docs/workflows/reference/tdd-plan) | Medium | Test-driven development planning (Red-Green-Refactor). |
| [Test-Fix-Gen](/en/docs/workflows/reference/test-fix-gen) | Medium | Generate tests + run automated test-fix execution cycle. |

### Brainstorm

| Workflow | Complexity | Description |
|----------|-----------|-------------|
| [Brainstorm](/en/docs/workflows/reference/brainstorm-auto-parallel) | High | Multi-role brainstorming to explore architecture and new features. |

### Issue Management

| Workflow | Complexity | Description |
|----------|-----------|-------------|
| [Issue:Discover](/en/docs/workflows/reference/issue-discover) | High | Batch issue discovery from the codebase (multi-perspective). |
| [Issue:Execute](/en/docs/workflows/reference/issue-execute) | High | Execute queued issues in parallel (worktree isolation optional). |
